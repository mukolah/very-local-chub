API Documentation

Token Management

Generate a bearer token to authenticate API requests. Store it securely — it grants full API access.

No token
—

✓ New token generated — copy it now, it won't be shown in full again.

Tokens are stored locally in your database. Rotating invalidates the previous token immediately.

Overview

All /api/v1/ endpoints require a bearer token in the Authorization header:

Authorization: Bearer <your-token>

Base URL: http://localhost:1488

Cards

GET/api/v1/cards

List cards with pagination and optional search/filter.

ParamTypeDescription
pageintegerPage number (default: 1)
querystringSearch query (comma-separated tags, prefix with - to exclude)
typestringbasic | tag | title | author
sortstringcreatedAt | lastActivityAt
curl -H "Authorization: Bearer TOKEN" \
     "http://localhost:1488/api/v1/cards?page=1&sort=lastActivityAt"

Response:

{
  "cards": [ { "id": 123, "name": "...", "author": "...", "topics": [...],
               "tokenCount": 1500, "quality_score": 4.0, "quality_bar": "⭐⭐⭐⭐☆ 4.0",
               "lewdity_score": null, "lewdity_bar": null,
               "story_score": 3.5, "story_bar": "📖📖📖½☆ 3.5",
               "createdAt": "Jan 01, 2024 12:00", "lastActivityAt": "...", ... } ],
  "page": 1, "total_pages": 12, "count": 1180
}

Same as /api/v1/cards but omits description and tagline per card. Useful when you only need identifiers, scores, and tags and want a lighter payload.

GET/api/v1/cards-full

List cards with the same filters as /api/v1/cards, but each card also includes raw_metadata (full parsed .json file), raw_data (character sheet from the embedded PNG chara block — description, first message, personality, scenario, creator notes, etc.), png_info (full chara JSON), and a structured scores object. Heavier than the standard list — use for bulk data export or agent workflows that need the complete source data.

ParamTypeDescription
pageintegerPage number (default: 1)
querystringSearch query (comma-separated tags, prefix with - to exclude)
typestringbasic | tag | title | author
sortstringcreatedAt | lastActivityAt
curl -H "Authorization: Bearer TOKEN" \
     "http://localhost:1488/api/v1/cards-full?page=1&sort=lastActivityAt"

Response:

GET/api/v1/cards/{id}

Full card details including character sheet data from the PNG.

curl -H "Authorization: Bearer TOKEN" \
     "http://localhost:1488/api/v1/cards/6194250"

Response: all fields from the list endpoint plus raw_data (character description, first message, creator notes, etc.) and png_info (full chara JSON).

PATCH/api/v1/cards/{id}

Update card scores and/or tags. All fields are optional.

FieldTypeDescription
qualityfloat 0–5Quality score (null to clear)
lewdityfloat 0–5Lewdity score (null to clear)
storyfloat 0–5Story score (null to clear)
topicsstring[]Replace full tag list
curl -X PATCH -H "Authorization: Bearer TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"quality": 4.5, "story": 3.0}' \
     "http://localhost:1488/api/v1/cards/6194250"

Tags

POST/api/v1/cards/{id}/tags/add

Add tags to a card without replacing existing ones.

curl -X POST -H "Authorization: Bearer TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"tags": ["fantasy", "adventure"]}' \
     "http://localhost:1488/api/v1/cards/6194250/tags/add"
POST/api/v1/cards/{id}/tags/remove

Remove specific tags from a card.

curl -X POST -H "Authorization: Bearer TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"tags": ["old-tag"]}' \
     "http://localhost:1488/api/v1/cards/6194250/tags/remove"

Agent Usage Example

A simple pattern for an AI agent to score cards:

# 1. Get a page of cards
GET /api/v1/cards?page=1&sort=createdAt

# 2. For each unscored card, read its full data
GET /api/v1/cards/{id}   # includes description, first_mes, creator_notes

# 3. Score it
PATCH /api/v1/cards/{id}
{ "quality": 3.5, "lewdity": 1.0, "story": 4.0 }

# 4. Optionally add/remove tags based on content
POST /api/v1/cards/{id}/tags/add
{ "tags": ["fantasy", "romance"] }

Rotate token?

This will invalidate your current token immediately. Any clients using it will stop working until you update them with the new token.
